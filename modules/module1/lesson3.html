Урок 3: HTML – Таблиці та форми для взаємодії

Таблиці (Tables)

HTML-таблиці використовуються для відображення табличних даних (даних, які мають рядки та стовпці). Не використовуйте таблиці для верстки макетів – для цього є CSS Flexbox та Grid!

Основні теги таблиць

  • <table>: Контейнер для всієї таблиці.
  • <tr> (Table Row): Визначає рядок у таблиці.
  • <th> (Table Header): Визначає клітинку заголовка (зазвичай відображається жирним і по центру).
  • <td> (Table Data): Визначає стандартну клітинку даних.

Приклад простої таблиці:

<table>
    <tr>
        <th>Назва товару</th>
        <th>Кількість</th>
        <th>Ціна</th>
    </tr>
    <tr>
        <td>Ноутбук</td>
        <td>1</td>
        <td>12000 грн</td>
    </tr>
    <tr>
        <td>Миша</td>
        <td>2</td>
        <td>350 грн</td>
    </tr>
</table>

Семантичні теги таблиць (для структуризації)

Для кращої семантики та доступності великих таблиць використовують:

  • <thead>: Групує вміст заголовка таблиці.
  • <tbody>: Групує основний вміст (тіло) таблиці.
  • <tfoot>: Групує вміст нижнього колонтитула (футера) таблиці.

Приклад з семантичними тегами:

<table>
    <thead>
        <tr>
            <th>День тижня</th>
            <th>Ранкові заняття</th>
            <th>Вечірні заняття</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Понеділок</td>
            <td>Математика</td>
            <td>Програмування</td>
        </tr>
        <tr>
            <td>Вівторок</td>
            <td>Фізика</td>
            <td>Спорт</td>
        </tr>
    </tbody>
    <tfoot>
        <tr>
            <td colspan="3">Розклад може змінюватися</td>
        </tr>
    </tfoot>
</table>

Об'єднання клітинок: colspan та rowspan

  • colspan: Об'єднує клітинку за стовпцями (горизонтально). Значення – кількість стовпців, які клітинка повинна займати.
  • rowspan: Об'єднує клітинку за рядками (вертикально). Значення – кількість рядків, які клітинка повинна займати.

Приклад об'єднання клітинок:

<table>
    <thead>
        <tr>
            <th rowspan="2">Категорія</th>
            <th colspan="2">Товар</th>
        </tr>
        <tr>
            <th>Назва</th>
            <th>Кількість</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Електроніка</td>
            <td>Смартфон</td>
            <td>50</td>
        </tr>
        <tr>
            <td></td> <!-- Порожня клітинка, бо "Електроніка" займає 2 рядки -->
            <td>Планшет</td>
            <td>30</td>
        </tr>
        <tr>
            <td rowspan="2">Одяг</td>
            <td>Футболка</td>
            <td>100</td>
        </tr>
        <tr>
            <td></td>
            <td>Джинси</td>
            <td>75</td>
        </tr>
    </tbody>
</table>

Порада: Важливо правильно розраховувати кількість клітинок при використанні colspan та rowspan, щоб структура таблиці була коректною.

Форми (Forms)

HTML-форми використовуються для збору даних від користувачів (наприклад, введення тексту, вибір опцій, завантаження файлів) та надсилання їх на сервер.

Тег <form> та його атрибути

  • <form>: Контейнер для всіх елементів форми.
  • action: URL-адреса, на яку відправлятимуться дані форми після її надсилання.
  • method: HTTP-метод для відправки даних.
    • GET: Дані форми додаються до URL як параметри запиту. Підходить для нечутливих даних (пошукові запити).
    • POST: Дані форми відправляються у тілі HTTP-запиту. Підходить для конфіденційних даних (паролі, особиста інформація), а також для великих обсягів даних.
<form action="/submit_form.php" method="POST">
    <!-- Елементи форми тут -->
</form>

Елементи вводу (Input Elements)

Найпоширеніший елемент форми – <input>. Його тип визначається атрибутом type.

  • type="text": Однорядкове текстове поле.
  • type="password": Поле для пароля (символи приховані).
  • type="email": Поле для електронної пошти (з базовою валідацією формату).
  • type="number": Поле для введення чисел.
  • type="checkbox": Прапорець (для множинного вибору).
  • type="radio": Перемикач (для вибору однієї опції з кількох, повинні мати однаковий name).
  • type="submit": Кнопка для відправки форми.
  • type="button": Звичайна кнопка (потребує JavaScript для функціональності).
  • type="date", type="color", type="range", type="file": Сучасніші типи для різних цілей.

Важливі атрибути для <input>:

  • name: Ім'я поля, яке буде використовуватися для ідентифікації даних при відправці на сервер. **Обов'язковий!**
  • value: Початкове значення поля або значення, яке буде відправлено (для радіокнопок, прапорців).
  • placeholder: Короткий текст-підказка, що відображається у полі, поки користувач не введе дані.
  • required: Булевий атрибут; поле є обов'язковим для заповнення.
  • readonly: Булевий атрибут; поле не може бути змінено користувачем.
  • disabled: Булевий атрибут; поле неактивне та не відправляється з формою.

Приклад різних полів <input>:

<form>
    <div>
        <label for="username">Ім'я користувача:</label>
        <input type="text" id="username" name="username" placeholder="Введіть ваше ім'я" required>
    </div>
    <div>
        <label for="useremail">Email:</label>
        <input type="email" id="useremail" name="useremail" placeholder="example@domain.com">
    </div>
    <div>
        <label for="userpass">Пароль:</label>
        <input type="password" id="userpass" name="userpass" minlength="8">
    </div>
    <div>
        <input type="checkbox" id="newsletter" name="newsletter" value="yes">
        <label for="newsletter">Підписатися на розсилку</label>
    </div>
    <div>
        <p>Ваша стать:</p>
        <input type="radio" id="male" name="gender" value="male">
        <label for="male">Чоловіча</label><br>
        <input type="radio" id="female" name="gender" value="female">
        <label for="female">Жіноча</label>
    </div>
    <div>
        <input type="submit" value="Відправити форму">
    </div>
</form>

Тег <label>

Тег <label> асоціює текстову мітку з елементом керування формою. Це покращує доступність, дозволяючи користувачам клікати на мітку, щоб активувати відповідний елемент вводу.

Використовуйте атрибут for у <label>, значення якого повинно співпадати з атрибутом id відповідного поля вводу.

<label for="myInput">Моє поле вводу:</label>
<input type="text" id="myInput" name="myInput">

Багаторядкове текстове поле (<textarea>)

Для введення великого обсягу тексту використовується <textarea>.

<div>
    <label for="comment">Ваші коментарі:</label>
    <textarea id="comment" name="comment" rows="5" cols="40" placeholder="Введіть ваш коментар..."></textarea>
</div>
  • rows: Визначає видиму кількість рядків.
  • cols: Визначає видиму ширину в символах.

Випадаючий список (<select> та <option>)

Для вибору однієї або кількох опцій зі списку використовується <select> разом з тегами <option>.

<div>
    <label for="country">Виберіть країну:</label>
    <select id="country" name="country">
        <option value="">-- Будь ласка, оберіть --</option>
        <option value="ua">Україна</option>
        <option value="us">США</option>
        <option value="ca">Канада</option>
    </select>
</div>

Атрибут value у <option> – це значення, яке буде відправлено на сервер.

Кнопки (Buttons)

Тег <button> є більш гнучким, ніж <input type="submit">, оскільки може містити HTML-контент (зображення, інші теги).

<button type="submit">Відправити</button>
<button type="reset">Очистити форму</button>
<button type="button">Звичайна кнопка</button>

Важливо: Завжди вказуйте атрибут type для <button>, щоб уникнути неочікуваної поведінки (за замовчуванням багато браузерів вважають його submit).

Практичне завдання: Розширений розклад та форма зворотного зв'язку

Створіть новий HTML-файл advanced_features.html і реалізуйте наступне:

  1. Налаштуйте базову структуру HTML5.
  2. Створіть розширену таблицю розкладу:
    • Додайте таблицю для розкладу на тиждень (5 робочих днів).
    • Використайте <thead>, <tbody>.
    • Перший стовпець – час (наприклад, 9:00, 10:00, 11:00).
    • Використайте colspan для об'єднання клітинок, якщо якесь заняття триває кілька годин, або rowspan для об'єднання днів, якщо є спільні елементи. Наприклад, об'єднайте клітинку "Обід" на всіх днях, або "Вихідні" на суботу та неділю.
    • Додайте мінімум 3-4 заняття на день.
  3. **Створіть форму зворотного зв'язку:**
    • Форма повинна мати атрибути action="#" та method="POST".
    • Поля:
      • Ім'я (text, обов'язкове, з placeholder).
      • Email (email, обов'язкове, з placeholder).
      • Телефон (tel, необов'язкове, з placeholder).
      • Тема повідомлення (випадаючий список <select> з 3-4 опціями, наприклад "Пропозиція", "Питання", "Скарга").
      • Ваше повідомлення (<textarea>, rows="7", з placeholder).
      • Прапорець для згоди з обробкою даних (обов'язковий).
    • Використайте <label> для кожного елемента форми та пов'яжіть їх за допомогою атрибутів for та id.
    • Кнопка "Відправити" (використайте <button type="submit">).

Збережіть файл, відкрийте у браузері та перевірте функціонал (хоча форма не буде відправлятися, оскільки action="#").
